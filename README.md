# Network Segmentation Lab
## Overview 
This project demonstrates the design of a segmented enterprise network for Cyber Waves
## Objectives
- Improve network security
- separate departments into different network zones
- Protect critical servers
- Control Network Traffic
## Architecture Design Process
### Step 1 - Analyze Business Requirements
- Identified the organization's departments and Business functions.
- Determined which systems and resources each department required.
### Step 2- Identify Security Requirements
- Identified critical assets that required additional protection
- Considered potential cybersecurity risks and how segmentation could reduce them. 
### Step 3- Design the Network Architecture
- Created a network diagram showing separate network segments for each department.
- Planned logical layout to immprove organization and security.
### Step 4- Apply Network Segmentation Principles
- Grouped systems with similar functions into separate network segments.
- Designed the architecture to isolate sensitive systems from general user networks.
- Planned the network to reduce unneccessary communication between segments.
### Step 5- Review the Design
- Reviewed the architecture to ensure it met the prject's security objectives.
- Confirmed the design supported improved network security and scalability.
